Installation

Required packages

The required packages for Hexagn to compile are:
  • G++ or MSYS2 MinGW for Windows (or Clang++ for MacOS Homebrew)
  • Make
If the required packages are not installed in your system, there are steps on how to install the required packages.
For linux, G++ and Make should be pre-installed by default, if not there are steps on how to install them.
For Windows, MSYS2 MinGW is not pre-installed, there are steps on how to install it.

NOTE

If you compiled Hexagn with MSYS2 MinGW, you should use it to compile any .hxgn files since running the .exe directly will lead to errors

Installing in Windows

To have the required packages, You could install MSYS2 MinGW which can be installed from https://www.msys2.org/.

Getting the compiler

After installing MSYS2 MinGW, get your desired version of the compiler from this page.
After installing the .zip file, extract it and go to the next step.

Installing Hexagn

After extracting the repository, go to the directory by using cd /c/[directory] in MSYS2 MinGW or access it via File explorer and opening with MSYS2 MinGW (.exe files are located at C:\msys2\ ).


Lets say the repository is located at C:\Users\NotAlternate\Downloads\Hexagn this is what you should run
$ cd /c/Users/NotAlternate/Downloads/Hexagn


Run this command when you are in the directory 
$ make


If you did all the process correctly, Hexagn should be installed


Check with 
$ hexagn --version




Installing in MacOS

To install the required packages, run the following commands in the terminal depending on your package manager

Using Homebrew 
$ brew update
$ brew install llvm # Clang++


Using MacPorts 
$ sudo port selfupdate
$ sudo port install gcc11 # GCC

Getting the compiler

To get the version of the repository, go to this page and download your desired version.
After installing the .zip file, extract it and go to the next step.

Installing Hexagn

After extracting the repository, go to the directory by using cd [directory] or access it via your file manager and opening in the terminal.

Run this command when you are in the directory 
$ make CXX=clang++  # Run this only if you installed via Homebrew
$ make              # Run this only if you installed via MacPorts


If you did all the process correctly, Hexagn should be installed

Check with 
$ hexagn --version


Installing in Linux

If the required packages are not pre-installed in your linux distro, run these command depending on your package manager:

APT - Debian, Ubuntu, Kali, etc 
$ sudo apt-get upgrade
$ sudo apt-get install g++


DNF/YUM - Fedora, RHEL, CentOS, OpenSUSE, etc 
$ sudo dnf upgrade
$ sudo dnf install gcc-c++

# For yum:
$ sudo yum upgrade
$ sudo yum install gcc-c++


PACMAN - Arch, Manjaro, etc 
$ sudo pacman -Syu
$ sudo pacman -Sy g++

Writting a basic program

This step is optional, this step is to show you how to use the compiler.

Programming

To make a basic Hello, World!, you can use your desired text editor.

In Windows you can use Notepad
In MacOS you can use TextEdit
In Linux you can use GNU Nano or Vim (if you are familiar with it)

You can paste the code below or program one yourself.


Basic Hello, World! 
int32 main() {
    println("Hello, World!");
}

Let's save the file as hello-world.hxgn and go to the next step.

Compiling

Note

If you compiled Hexagn using MSYS2 MinGW, open it and go to the directory.


Lets say the file is located at C:\Users\NotAlternate\Documents\hello-world.hxgn this is what you should run
$ cd /c/Users/NotAlternate/Documents


Let's compile the file by using the compiler and save it as hello-world.urcl
(Note that the compiler can only compile .hxgn and save it as .urcl ).


Compiling the file
$ hexagn hello-world.hxgn -o hello-world.urcl

If you saved it on a different filename, you can use hexagn input-filename.hxgn -o output-filename.urcl

Running

This step needs something to run URCL, you can use EURCL (Emulator for URCL) or URCX (URCL Explorer).

EURCL can be found in this page.
URCX can be found in this page.


Running the file in EURCL
$ eurcl hello-world.urcl

You can add some flags to the command line to change the behavior of the emulator.

Running in URCX (URCL Explorer)

Go to this page and open/paste the file you want to run.
Click Compile and your .urcl file will run.
